Genco Shopify Migration
Meta Ads Safe Cutover Plan

This page guides the move from the current store to a new Shopify build while protecting Meta Ads continuity, attribution quality, and conversion performance.

Current Store https://gencostores.com/
Main Goal Zero paid-traffic disruption
Approach URL continuity + tracking continuity

Core Rules

  • Keep the same live domain on launch day (`gencostores.com`).
  • Keep high-spend landing page paths unchanged where possible (`/products/...`, `/collections/...`, `/pages/...`).
  • If a URL must change, set a 301 redirect before launch and test with `utm_` and `fbclid` parameters.
  • Use the same Meta Pixel ID and enable Shopify Conversions API in the new store.

Why It Matters

  • Stable URLs reduce ad disapprovals and avoid last-minute campaign edits.
  • 301 mapping preserves link equity and lowers traffic leakage from old links.
  • Pixel + CAPI continuity keeps attribution comparable before vs after migration.
  • Correct launch sequencing protects delivery during algorithm re-learning windows.
Client Dev Team

Owns Technical Migration

  • URL mapping and 301 redirect import.
  • Theme QA for mobile speed and checkout flow.
  • Event firing checks on all critical templates.
Admireworks

  • Identify top ad landing pages to preserve first.
  • Validate Pixel and CAPI event quality post-cutover.
  • Monitor CPA, CVR, and event-match quality after launch.

Shared

Joint Go-Live Decisions

  • Final go/no-go check before DNS switch.
  • Launch window selection (lower traffic hours).
  • Rollback trigger thresholds and response owner.
  • Daily checkpoint in first 72 hours.
1

Pre-Launch (7-10 days before)

Build confidence before traffic touches the new store
  • Export current sitemap and list top paid landing URLs.
  • Create old-to-new URL mapping sheet and mark redirect owner.
  • Set Meta integration in Shopify with maximum data sharing and verify events in test mode.
  • Test full funnel from ad click to purchase with query parameters preserved.
2

Launch Window (Cutover day)

Protect active campaigns while DNS and routes switch
  • Switch primary domain to new store and confirm canonical domain behavior.
  • Publish redirects and run spot checks on highest-spend ad URLs.
  • Confirm `ViewContent`, `AddToCart`, `InitiateCheckout`, and `Purchase` in Meta Events Manager.
  • Keep budget stable; avoid major campaign edits in the same hour unless critical.
3

First 72 Hours

Catch regressions early and stabilize performance
  • Track CPC, CTR, CVR, CPA, and purchase event volume vs previous baseline.
  • Fix any redirect misses immediately; prioritize high-spend URLs first.
  • Review event quality (deduplication, match quality, currency/value accuracy).
  • If severe drop appears, pause only affected ads and use rollback plan for impacted routes.

Go/No-Go Decision Gates

Go

Critical events fire correctly, top landing URLs resolve correctly, checkout works on mobile, and no major attribution anomalies in the first live checks.

No-Go / Hold

Missing purchase events, redirect failures on paid URLs, checkout blockers, or major mismatch between ad clicks and on-site sessions.
